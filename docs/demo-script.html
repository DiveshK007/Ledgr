Pre-flight checklist

curl localhost:11434/api/version responds, and gemma4:12b shows in ollama list.

Known issues & recovery

FIXED

Pricing agent returned empty text

Root cause: two sequential tool calls (one per perishable item) exhausted the model's default generation budget. Fixed by raising num_ctx/num_predict and adding a no-tools retry if the final message is still empty. Confirmed passing.

LIMIT

Cash Flow's shortfall path isn't typed-query-reachable

The planner doesn't parse a rupee amount out of free text yet, so a typed "can I afford ₹30,000" won't pass pending_expense through. Use the CLI command in §06 Part B for that beat.

SLOW

Supplier can look "stuck" for minutes

Three sequential vision calls on a 12B model, CPU-only. It's working — check the trust panel if you ever doubt it; a decision only logs there once the agent actually finishes, even if a browser request times out first.

LIMIT

403 "AirTunes" instead of your API response

macOS AirPlay Receiver owns port 5000 over IPv6. The frontend proxy is already pointed at 127.0.0.1:5000 to avoid it — if you're curling manually, do the same.
